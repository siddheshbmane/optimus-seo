## 1. BRAND STORY & ORIGIN

### 1.1 The Founder's "Why"

Running an SEO agency means drowning in execution. Every new client adds hours of keyword research, competitor analysis, content creation, backlink prospecting, directory submissions, technical audits, and monthly reporting. The tools exist — Ahrefs, SEMrush, Moz — but they only tell you *what* to do. They never *do* it.

The founder experienced this firsthand: a growing client roster, a team stretched thin, and the constant tension between taking on more work and maintaining quality. Hiring more SEO executives was expensive and slow. Training them took months. Quality was inconsistent. And the most frustrating part? 85% of the work was repetitive execution that followed predictable patterns — exactly the kind of work AI agents should handle.

The realization was simple but powerful: **What if the entire execution layer of an SEO agency could be automated?** Not just analysis dashboards. Not just content suggestions. Actual execution — from generating a sales pitch deck for a prospect, to researching keywords, to writing content that passes quality review, to submitting backlinks, to compiling monthly reports. All powered by real SEO data, all orchestrated by AI agents, all approved by humans at critical checkpoints.

That's how Optimus SEO was born — not as another SEO tool, but as an **AI-powered SEO operations platform** that lets strategists and sales people run an entire department.

### 1.2 Mission Statement

**To eliminate the execution bottleneck in SEO by replacing manual, repetitive work with intelligent AI agents — so SEO professionals can focus on strategy, creativity, and client relationships.**

### 1.3 Vision Statement (5-10 Year Horizon)

**To become the operating system for SEO agencies worldwide — where every agency, regardless of size, can deliver enterprise-grade SEO at scale through AI-powered automation, making exceptional SEO accessible to every business.**

### 4.2 Writing Dos and Don'ts

| ✅ DO | ❌ DON'T |
|-------|----------|
| Use precise numbers and data ("85% autonomous", "60+ metrics") | Use vague superlatives ("the best", "amazing", "revolutionary") |
| Be direct and confident ("Optimus handles this", not "Optimus can help with this") | Be wishy-washy or hedge ("might", "could potentially", "we think") |
| Use active voice ("Agents research keywords" not "Keywords are researched by agents") | Use passive voice or corporate jargon ("leverage", "synergize", "paradigm shift") |
| Show personality with dry wit and clever phrasing | Force humor, use puns, or be cheesy |
| Explain technical concepts clearly without dumbing them down | Oversimplify to the point of inaccuracy |
| Address the reader directly ("your agency", "your clients") | Be generic ("agencies", "users", "customers") |
| Use short, punchy sentences for impact. Then elaborate. | Write long, winding sentences that lose the reader |
| Acknowledge what competitors do well, then show why we're different | Trash-talk competitors or make unsubstantiated claims |

## 7. COLOR PALETTE

### 7.1 Design Rationale

The color palette follows a **GitHub-inspired "Tech" aesthetic** — clean, developer-friendly, and content-focused. The monochrome foundation keeps data-dense SEO dashboards readable, while the coral accent (`#FD8C73`) provides warmth and draws attention to key actions without overwhelming the interface. This approach prioritizes **information density and readability** over visual flair — exactly what SEO professionals need when analyzing thousands of keywords and metrics.

The palette avoids purple/violet AI gradients (banned by blueprint) and generic corporate blue. It also avoids the orange-heavy palettes of Ahrefs/SEMrush, creating a distinct identity.

## 8. TYPOGRAPHY

### 8.1 Font Selection

| Role | Font | Weight Range | Usage in wireframe.pen |
|------|------|-------------|------------------------|
| **Headings** | **Inter** (Variable) | 600, 700 | Page titles, section headings, card titles, stat values |
| **Body** | **Inter** (Variable) | 400 (normal), 500 (medium) | Body text, descriptions, table cells, form labels |
| **Technical / Mono** | **JetBrains Mono** (Variable) | 400, 500, 600, 700 | Logo wordmark, stat numbers, badge text, table data, API keys, code, labels, timestamps |

> **Why two fonts?** The wireframe.pen design uses Inter as the workhorse for all readable content, and JetBrains Mono extensively for anything "data-like" — stat card numbers, badge labels, table row data, timestamps, and the logo itself. > **JetBrains Mono usage is broader than typical "code" font.** In the wireframe.pen design, JetBrains Mono appears in:
> - Logo wordmark ("Optimus SEO" — JetBrains Mono 700)
> - Stat card values (e.g., "2,847", "94.2%")
> - Badge text (e.g., "Live", "Draft", "Pending")
> - Table cell data (keyword volumes, rankings, URLs)
> - Timestamps and dates
> - API keys and technical identifiers
> - Navigation item labels (in some contexts)

### 8.4 Font Loading Strategy

```typescript
// next.config.ts — using next/font for optimal loading
import { Inter, JetBrains_Mono } from 'next/font/google';

export const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-inter',
});

export const jetbrainsMono = JetBrains_Mono({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-mono',
});
```

```css
/* Tailwind CSS config mapping */
fontFamily: {
  sans: ['var(--font-inter)', 'system-ui', 'sans-serif'],
  mono: ['var(--font-mono)', 'JetBrains Mono', 'Menlo', 'monospace'],
}
```

> **Performance note:** Both fonts are loaded as variable fonts via `next/font`, which means a single file per font (~100KB Inter + ~90KB JetBrains Mono). The `display: 'swap'` strategy ensures text is visible immediately with system fonts, then swaps to the custom fonts once loaded. Total font payload: ~190KB (cached after first load).

**Geometric / Flat with depth**

- Style: Clean geometric shapes with subtle gradients and shadows
- Use for: Feature explanations, empty states, onboarding, error pages
- Color: Use brand palette (coral accent + monochrome neutrals + chart colors)
- Avoid: Hand-drawn, 3D renders, overly detailed illustrations
- Inspiration: Linear, Vercel, GitHub illustration styles

### 9.3 Icon Style

**Outlined (Lucide Icons)**

- Primary icon set: [Lucide](https://lucide.dev/) — consistent, open-source, React-native
- Style: 1.5px stroke, rounded caps and joins
- Size: 16px (inline), 20px (default), 24px (large), 32px (feature icons)
- Color: Inherit from text color (`text-secondary` default, coral `#FD8C73` for active/accent)
- Custom icons: Only when Lucide doesn't have an appropriate option — maintain same stroke weight and style

### 9.4 AI-Generated Imagery Guidelines

- **When to use:** Abstract backgrounds, decorative elements, marketing page hero visuals
- **When NOT to use:** Product screenshots (must be real), team photos, customer photos
- **Style guide:** If AI-generated, must match the geometric/abstract style — no photorealistic AI images
- **Disclosure:** Not required for abstract/decorative use, but never claim AI images are real photos
